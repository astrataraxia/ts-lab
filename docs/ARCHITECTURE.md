# 프로젝트 아키텍처

이 프로젝트는 화면 단위 기능을 `app`에 두고, 공통 표현과 전역 스타일을 `styles`에 두는 얇은 TypeScript 웹 앱입니다. 특정 화면이나 실험의 이름은 아키텍처 문서의 일부가 아닙니다.

## 디렉터리 역할

```text
src/
├─ main.ts                 # 앱 진입점과 초기 조립
├─ app/
│  ├─ pages/               # 화면 단위 기능
│  │  └─ <page>/           # 페이지별 구현과 페이지 전용 모듈
│  └─ ...                  # 여러 페이지가 공유하는 앱 모듈
└─ styles/                 # 전역 스타일과 공통 시각 규칙
```

실제 페이지가 추가되면 `app/pages` 아래에 페이지 단위로 구성합니다. 페이지 내부의 컴포넌트, 데이터, 상태, 서비스 등은 해당 페이지의 책임에 속하며, 여러 페이지에서 반복해서 사용될 때만 `app`의 공통 모듈로 승격합니다.

## 실행과 의존 방향

```text
main.ts
  ↓
app/pages 및 공통 app 모듈
  ↓
DOM

styles
  ↓
앱의 시각 표현
```

`main.ts`는 앱을 시작하고 필요한 최상위 모듈을 조립합니다. 페이지는 자신의 화면 동작을 소유하며 진입점의 구현 세부사항에 의존하지 않습니다. 스타일은 앱의 시각적 표현을 담당하고, 페이지의 업무 로직을 소유하지 않습니다.

## 책임 규칙

- `main.ts`: 앱 진입, 전역 초기화, 최상위 조립
- `app/pages`: 화면 단위 기능과 그 화면에 종속된 구현
- `app`의 공통 모듈: 둘 이상의 페이지에서 실제로 공유되는 기능
- `styles`: 전역 스타일, 디자인 토큰, 공통 레이아웃과 시각 규칙

새 기능은 먼저 자신이 속한 페이지 안에서 작게 시작합니다. 공유가 확인되지 않은 코드는 미리 공통 모듈로 만들지 않습니다.

## 문서 변경 기준

페이지나 실험의 추가·삭제·이름 변경은 이 문서를 수정할 사유가 아닙니다. 디렉터리 경계, 계층의 책임, 의존 방향이 바뀌는 경우에만 이 문서를 갱신합니다.
