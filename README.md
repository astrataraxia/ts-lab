# Studio/01

작은 인터랙티브 실험을 모아두는 TypeScript 프로젝트입니다.

## 시작하기

```powershell
npm install
npm run dev
```

## 주요 기능

- Home: 프로젝트 진입 허브
- Sound Canvas: 오디오 데이터를 이용한 Canvas 시각화
- ORD Helper: TMO.GG 연동 원랜디 추천 도우미

## TMO.GG 연동

ORD Helper는 TMO.GG 데스크톱의 로컬 응답 API를 읽습니다.

- 데이터: `http://127.0.0.1:25625/datas`
- 조합 카탈로그: 프로젝트에 포함된 [`public/data/ord-helper/assets.json`](public/data/ord-helper/assets.json)
- 보조 Lua: [`public/scripts/ord-advisor.lua`](public/scripts/ord-advisor.lua)

조합 카탈로그는 TMO.GG 빌드 헬퍼 36572 페이지의 정리된 로컬 사본입니다. 28개 그룹과 305개 항목을 포함하며, `왜곡됨`, `세라핌`, 물딜·마딜·스턴 세부 분류도 포함합니다. 추천에 필요한 `id`, 정리된 `name`, `codes`, `stuffs`, `abilities`, `desc`, `selectWisp`만 보존하고 이미지·명령어·중복 그룹명·빈 필드는 제외했습니다. 게임을 실행할 때마다 외부에서 조합 데이터를 다시 받지 않으며, 원본 페이지가 변경되었을 때만 수동으로 갱신하면 됩니다.

추천은 눈앞의 단일 조합 가능성만 비교하지 않고, 왜곡됨·영원·제한됨·불멸·초월·세라핌·신비함 목표에서 조합 경로를 역산합니다. 현재 보유 유닛의 물딜·마딜 축을 유지하면서 방깍·아머브레이크·마방깍·버프·실딜·라인딜·단일딜·끝딜·보스딜·광폭딜 역할을 조합 단위로 평가합니다. 최종 목표가 딜러인지 버퍼인지에 따라 부족한 역할을 보완하고, 이감이 맞춰진 뒤 스턴 1 이상을 확보하는 순서를 우선합니다. 최상위 목표가 마나 스킬인지 체력 스킬인지 판별해 마나젠·체젠 지원도 별도로 맞춥니다. 전용 아이템이나 특수 강화 아이템은 필수/선택 강화로 구분해 보유 여부에 따라 목표 성능 점수를 보정하고, 위습을 포함한 하위 재료와 자원 부족분을 함께 계산합니다.

실제 게임 상태는 TMO.GG 데스크톱의 `/datas` 로컬 엔드포인트에서만 1초 간격으로 읽습니다. 조합 카탈로그는 프로젝트에 포함된 정적 파일이며, 수신한 유닛 ID는 메모리 Map으로 즉시 이름에 매칭합니다. 추천 목록에서는 `흔함`·`안흔함`을 제외하고 특별함·희귀함·전설·히든·최상위 유닛으로 이어지는 조합 경로를 트리로 표시합니다.
